============================================================
TRIGGERS Y FUNCIONES CLAVE
============================================================

fn_set_updated_at()
- Actualiza updated_at con now() en tablas que lo usan.

fn_rol_actual()
- Devuelve el rol del usuario actual consultando perfiles por auth.uid().

fn_on_ot_waiting_parts()
- Trigger after update on ordenes_trabajo.
- Al pasar OT a waiting_parts, genera PR automatica.

fn_on_recepcion_conforme()
- Trigger after insert on recepciones_oc_detalle.
- Si conformidad = conforme, actualiza stock e inserta movimiento entrada_compra.
- Si no conforme, penaliza proveedor.

fn_on_salida_consumo()
- Trigger after insert on movimientos_inventario.
- Actualiza stock segun salida_consumo, ajuste_positivo y ajuste_negativo.

fn_generar_orden_venta(p_ot_id, p_costo_servicio)
- Crea ordenes_venta y ordenes_venta_detalle al cerrar OT.
- Estado inicial actual: creada.
- No crea OV duplicada si ya existe.

fn_actualizar_orden_venta_costo_servicio(p_ov_id, p_costo_servicio)
- Actualiza costo_servicio.
- Recalcula costo_total.
- Cambia estado a con_costo_servicio.

fn_cancelar_orden_venta(p_ov_id)
- Cancela una OV solo si esta en con_costo_servicio.

============================================================
RLS / POLICIES
============================================================

La base usa RLS por rol mediante fn_rol_actual().

Policies relevantes:
- perfiles: solo propio perfil o administrador
- sesiones_log: propio usuario o administrador
- repuestos: administrador, logistica
- proveedores: administrador, logistica
- ordenes_trabajo: administrador, asesor_servicio, tecnico
- diagnosticos_ot: administrador, tecnico
- movimientos_inventario: administrador, almacenero, tecnico
- rfq: administrador, logistica
- aprobaciones_proveedor: administrador, logistica
- ordenes_compra: administrador, logistica; gerencia solo puede actualizar si estado = pendiente_aprobacion
- recepciones_oc: administrador, almacenero
- recepciones_oc_detalle: administrador, almacenero
- alertas: administrador, almacenero, logistica, gerencia, asesor_servicio, tecnico
- ordenes_venta: select para roles operativos; insert/update para administrador, asesoria/logistica segun flujo

Policies agregadas por migraciones recientes:
- categorias: insert/update para administrador, logistica
- parametros_inventario: insert/update para administrador, logistica
- ot_repuestos_requeridos: insert/update/delete para administrador, tecnico
- requisiciones_compra: insert/update para administrador, almacenero, logistica
- pr_detalle: insert/update/delete para administrador, almacenero, logistica
- historial_consumo: insert para administrador, tecnico, logistica

============================================================
NOTAS
============================================================

- UTC se usa en timestamptz; Lima opera en UTC-5 en la capa de presentacion.
- OT -> PR -> RFQ -> OC -> Recepcion -> OV es el flujo principal.
- Las OVs nacen al cerrar OT y luego reciben costo de servicio en un paso aparte.
- El archivo se debe mantener sincronizado con nuevas migraciones, no editando el SQL base original.
